## Overview

`ISO639_2B` enumerates only the **bibliographic (B) form** of ISO 639-2 alpha-3 codes вЂ” the ~20 languages that have two codes: a B form (historical, librarianship-oriented) and a T form (terminologic, preferred by IETF BCP 47).

## Specific API

| Signature | Description |
|---|---|
| `getBibliographicToTerminologicMap(): array<string, string>` | Returns the complete Bв†’T map (lazy-cached) |
| `toTerminologic(string $code): ?string` | Converts a B code to its T equivalent, or `null` if unknown |

> рџ’Ў The map is exposed as a **static method** rather than a class constant to avoid `ConstantsTrait::includes()` traversing its values (otherwise `ISO639_2B::includes('fra')` would return `true` because `fra` is a value of the map, which is semantically wrong).

## Examples

```php
use org\iso\ISO639_2B;

ISO639_2B::FRE;                                    // 'fre'
ISO639_2B::includes('fre');                        // true
ISO639_2B::includes('fra');                        // false вЂ” fra is T (see ISO639_2)

ISO639_2B::toTerminologic('fre');                  // 'fra'
ISO639_2B::toTerminologic('ger');                  // 'deu'
ISO639_2B::toTerminologic('eng');                  // null вЂ” no B variant

ISO639_2B::getBibliographicToTerminologicMap();    // ['alb' => 'sqi', 'arm' => 'hye', ...]
```

### Use case вЂ” normalize a legacy code to BCP 47

```php
use org\iso\ISO639_2;
use org\iso\ISO639_2B;

function normalizeAlpha3( string $code ): ?string
{
    // If it's a known B code, convert it to T
    if ( $canonical = ISO639_2B::toTerminologic( $code ) )
    {
        return $canonical;
    }
    // Otherwise, check it's in ISO639_2 (canonical)
    return ISO639_2::includes( $code ) ? $code : null;
}

normalizeAlpha3('fre');   // 'fra' (B в†’ T)
normalizeAlpha3('fra');   // 'fra' (already T)
normalizeAlpha3('eng');   // 'eng' (unique T)
normalizeAlpha3('zzz');   // null
```
