## Vue d'ensemble

`ISO639_2B` énumère uniquement la **forme bibliographique (B)** des codes ISO 639-2 alpha-3, soit ~20 langues qui possèdent deux codes : un B (historique, bibliothéconomique) et un T (terminologique, préféré par IETF BCP 47).

| Code B | Code T | Langue |
|---|---|---|
| `alb` | `sqi` | Albanian |
| `arm` | `hye` | Armenian |
| `baq` | `eus` | Basque |
| `bur` | `mya` | Burmese |
| `chi` | `zho` | Chinese |
| `cze` | `ces` | Czech |
| `dut` | `nld` | Dutch; Flemish |
| `fre` | `fra` | French |
| `geo` | `kat` | Georgian |
| `ger` | `deu` | German |
| `gre` | `ell` | Modern Greek (1453-) |
| `ice` | `isl` | Icelandic |
| `mac` | `mkd` | Macedonian |
| `mao` | `mri` | Maori |
| `may` | `msa` | Malay |
| `per` | `fas` | Persian |
| `rum` | `ron` | Romanian; Moldavian; Moldovan |
| `slo` | `slk` | Slovak |
| `tib` | `bod` | Tibetan |
| `wel` | `cym` | Welsh |

> ℹ️ Pour la forme canonique (T-ou-unique), voir [`ISO639_2`](ISO639_2.md).

## API spécifique

| Signature | Description |
|---|---|
| `getBibliographicToTerminologicMap(): array<string, string>` | Renvoie la map complète B→T (lazy-cached) |
| `toTerminologic(string $code): ?string` | Convertit un code B en son équivalent T, ou `null` si inconnu |

> 💡 La map est exposée comme **méthode statique** plutôt que constante de classe pour éviter que `ConstantsTrait::includes()` ne traverse ses valeurs (sinon `ISO639_2B::includes('fra')` retournerait `true` car `fra` est une valeur du map, ce qui est sémantiquement faux).

## Exemples

```php
use org\iso\ISO639_2B;

ISO639_2B::FRE;                                    // 'fre'
ISO639_2B::includes('fre');                        // true
ISO639_2B::includes('fra');                        // false — fra est T (voir ISO639_2)

ISO639_2B::toTerminologic('fre');                  // 'fra'
ISO639_2B::toTerminologic('ger');                  // 'deu'
ISO639_2B::toTerminologic('eng');                  // null — pas de variant B

ISO639_2B::getBibliographicToTerminologicMap();    // ['alb' => 'sqi', 'arm' => 'hye', ...]
```

### Cas d'usage — normaliser un code legacy vers BCP 47

```php
use org\iso\ISO639_2;
use org\iso\ISO639_2B;

function normalizeAlpha3( string $code ): ?string
{
    // Si c'est un code B connu, on le convertit en T
    if ( $canonical = ISO639_2B::toTerminologic( $code ) )
    {
        return $canonical;
    }
    // Sinon, on vérifie qu'il est bien dans ISO639_2 (canonique)
    return ISO639_2::includes( $code ) ? $code : null;
}

normalizeAlpha3('fre');   // 'fra' (B → T)
normalizeAlpha3('fra');   // 'fra' (déjà T)
normalizeAlpha3('eng');   // 'eng' (T unique)
normalizeAlpha3('zzz');   // null
```
